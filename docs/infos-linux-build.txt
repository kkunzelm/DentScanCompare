PROJECT SOURCE CHANGES NEEDED (not yet done)
---------------------------------------------
The current CMakeLists.txt uses Qt5 — must be migrated to Qt6 before either platform build works.
Key changes required in CMakeLists.txt:
  - find_package(Qt5 5.15 ...) → find_package(Qt6 6.x REQUIRED COMPONENTS Widgets Concurrent PrintSupport OpenGL OpenGLWidgets)
  - target_link_libraries: Qt5::* → Qt6::*
  - Qt6 adds OpenGLWidgets as a separate component (QOpenGLWidget moved out of Qt6::OpenGL)

Key changes required in src/CMakeLists.txt:
  - Qt5::Widgets Qt5::Concurrent Qt5::PrintSupport Qt5::OpenGL
    → Qt6::Widgets Qt6::Concurrent Qt6::PrintSupport Qt6::OpenGL Qt6::OpenGLWidgets

Potentially affected source files (scan for Qt5-specific API):
  - src/main.cpp           uses QVTKOpenGLNativeWidget.h (check VTK header path for Qt6 build)
  - src/visualization/VTKMeshWidget.cpp  uses QVTKOpenGLNativeWidget.h
  MSVC-specific issues to watch for:
  - M_PI, M_PI_2 etc. not defined by default under MSVC → use std::numbers::pi (C++20) or #define _USE_MATH_DEFINES
  - POSIX functions: no direct impact found in grep — all file I/O uses std::filesystem and Qt classes
  - __attribute__ not supported by MSVC — not found in codebase, probably safe
  - min/max macros: Windows.h defines min/max as macros — add #define NOMINMAX before any Windows header if needed
